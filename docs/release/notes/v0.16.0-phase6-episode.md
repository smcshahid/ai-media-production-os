## Summary

Phase 6 introduces the **Episode Model Pilot**: projects can own numbered episodes, pipeline runs bind to `episode_id`, and exports use **manifest v4** with `episodes/episode_XX/` layout. Legacy runs without episodes continue the v1–v3 export ladder unchanged.

---

## Highlights

- `episodes` table + nullable `pipeline_runs.episode_id` (Alembic **0005**, additive only)
- `POST /episodes`, episode-scoped pipeline start and status
- Export manifest v4 with episode paths and narration sidecars
- Episode selector on dashboard; run history episode fields
- US-V07 Olares attestation PATH A–E PASS (C1 supplemental)

---

## Migration

```bash
alembic upgrade head   # applies 0005
```

No data backfill required. Existing projects operate unchanged until episodes are created.
